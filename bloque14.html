Bloque 14

14. Cronómetro

Mide y acumula el tiempo durante el cual una entrada permanece activa; almacena el valor hasta reset.

← Inicio Ir a descarga
Tipo: Cronómetro Entradas: En, Lap, R, Par Salida: AQ

Descripción

El bloque Cronómetro permite medir y acumular el tiempo durante el cual una señal de entrada (En) permanece activa. El cronómetro incrementa su cuenta mientras En esté en nivel alto y mantiene el valor acumulado aunque En sea desactivada, hasta que se aplique un reset (R) o se borre manualmente según la lógica del programa. Dispone además de una entrada Lap para capturar tiempos parciales (pausa/retención) sin perder la medición acumulada.

Pinout y descripción de conexiones

Entrada (En)
En (Enable) es la entrada principal que inicia o continúa la medición. Cuando En pasa de 0 a 1, el cronómetro comienza o retoma el conteo según el modo; al pasar a 0 puede detener la acumulación pero el valor queda almacenado hasta reset.
Entrada (Lap)
Lap captura un tiempo parcial: un flanco ascendente (0→1) congela temporalmente la salida AQ para mostrar el tiempo de pausa mientras el contador puede seguir o no dependiendo de la configuración. Un flanco descendente (1→0) restaura la salida al estado normal.
Entrada (R)
Reset: una señal en R borra el tiempo acumulado actual y reinicia cualquier valor de lap o pausa al valor inicial (0).
Parámetro (Par)
Permite seleccionar la base de tiempo (horas, minutos, segundos, centésimas) y ajustar precisión y límites del cronómetro.
Salida (AQ)
AQ emite el valor del tiempo medido (o el tiempo de pausa cuando Lap está activo). Esta salida puede conectarse a visualizaciones, registros o comparadores en el programa para actuar según el tiempo acumulado.

Notas de uso

  • Usar entrada Lap para capturar vueltas o tiempos intermedios sin perder la medición total.
  • Combinar con bloques de comparación para activar salidas al alcanzar determinado tiempo.
  • Verificar la base de tiempo (Par) para asegurar la precisión requerida en la aplicación.
